# 절차적 레벨 디자인 자동화 에디터 (ProcGen)

**프로젝트**: 재조합 (블루이글루) 게임 개발 · 인디 게임 개발 · 2026.06

## 개요

익스트랙션 슈터(탈출 슈터, Tarkov·Dark and Darker 계열)용 레벨을 시드 하나와 몇 가지 설정만으로 자동 생성하는 Unity 에디터 툴입니다. 레벨 디자인 반복 작업을 자동화하기 위해 AI를 활용해 구현했고, 실제 사용성을 검증하기 위한 테스트 파이프라인까지 함께 만들었습니다.

## 무엇을 하는 도구인가

- 버튼 한 번으로 **지형(산·평지·길)** 과 그 위의 **시작점·탈출구·몬스터·보상·보스**가 자동 배치
- **같은 시드 + 같은 설정 = 항상 같은 결과** (결정론적 생성). 마음에 든 맵을 시드 숫자로 재현 가능
- "플레이어가 달려서 60~120초면 탈출구 도착" 같은 **플레이 시간 기준**으로 맵 크기를 자동 계산 (TimeBased Scale)
- 만든 던전이 **실제로 끝까지 갈 수 있는지(막힌 길 없는지)** 를 도구가 자동 검사

## 구현 방식 (AI 활용)

Claude Code를 활용해 노이즈 기반 절차적 생성 파이프라인을 설계·구현했습니다. 단순 코드 생성이 아니라, 레벨 디자인 요구사항을 단계별 모듈로 분해하고 각 단계를 ScriptableObject 기반 설정 에셋으로 조립하는 구조로 만들었습니다.

- **필드 생성 (Stage A)** — 높이(Height)·위험도(Risk)·보상 분포(Loot) 노이즈 레이어를 합성해 지형과 배치 확률의 기반을 생성
- **구역 분할 (Region Partitioner)** — 맵을 외곽/중간/핵심 구역으로 나누고, 위험도가 높은 구역은 지형을 평지로 펴 전투 공간으로 조성
- **배치 (Interpreters)** — 시작점·탈출구·몬스터·엘리트·보스·보상·엄폐물을 규칙 기반으로 배치, 시작점↔탈출구 사이에는 통로를 자동 생성
- **지형화 (Terrain Builder)** — 높이 정보를 실제 Unity Terrain / NavMesh로 변환
- 결과물은 `Ctrl+Z`로 되돌릴 수 있도록 에디터 Undo와 통합, Target Root 아래로 정리해 씬에 생성

## 실사용성 연구 — 자동 검증(Validator) 시스템

도구가 만든 레벨이 실제로 플레이 가능한지를 사람이 매번 확인하지 않아도 되도록, 4종의 자동 검증기를 구현해 생성-검증 루프를 만들었습니다.

| Validator | 검사 내용 |
|---|---|
| **Reachability** | 시작점에서 탈출구까지 실제로 도달 가능한지 (막힌 길 검사) |
| **Balance** | 시작점↔탈출구, POI 간 거리 분포가 적절한지 |
| **Pacing** | 이동 시간이 목표 플레이 타임(예: 60~120초) 범위에 들어오는지 |
| **Affordance** | 캐릭터 이동 능력 기준으로 지형을 실제로 통과할 수 있는지 |

검증에 실패하면 설정된 횟수(Max Regeneration Attempts)만큼 시드를 바꿔가며 자동 재생성해, "생성 → 검증 → 실패 시 재생성"이 도구 안에서 자동으로 돌도록 했습니다. 샘플 실행 시 콘솔에 `[SampleTest] ... validation=PASS`가 출력되도록 테스트 진입점(`Tools ▸ ProcGen ▸ Sample ▸ Setup and Run Test`)도 제공해, 도구의 동작을 5초 만에 재현·검증할 수 있게 만들었습니다.

## 에디터 UX

코드를 몰라도 메뉴 클릭과 값 입력만으로 레벨을 만들 수 있도록 `Tools ▸ ProcGen ▸ Dungeon Preview` 창을 구현했습니다.

- 높이/위험/보상/구역 4종 지도 **미리보기**
- **Randomize Seed** 로 원하는 형태가 나올 때까지 반복
- **Generate Layout → Validate Layout → Apply to Scene** 순의 명확한 워크플로
- **Seed Gallery** 로 마음에 든 시드를 저장/복원
